2016-07-12新微商修改发货方式及更新记录

Jul 15, 2016

07-12 测试服务器 本地版本号 1.0.2

个人资料修改方式，
实名认证；
买家无效订单列表，买家可以取消订单；
卖家待发货未转采购的订单可以置为无效订单；
卖家无效订单列表，列表可以将无效订单恢复为有效订单；
投诉上级

买家扫码收货

买家确认收货

买家收货

canReceiveGood
0 不能确认收货
1 可以收货，不需要扫码
2 可以收货，必须扫码

买家收货接口
version == 1 使用老接口
// base.url.buyerHandleOrder_url+”?a=buyer_confirm_received”

version == 2 使用新接口
// base.url.doOrder_url+”?a=buyer_received”

======================

卖家确认发货

卖家下一步扫码发货

canDeliverGood
0 不能点确认发货
1 确认发货不需要扫码
2 确认发货需要扫码

卖家发货接口
version == 1 使用老接口
// base.url.sellerHandleOrder_url+”?a=seller_confirm_deliver”
version == 2 使用新接口
// base.url.doOrder_url+”?a=seller_deliver”
